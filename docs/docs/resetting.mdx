# Resetting the form & setting values

## Resetting the form

The form can be reset via the native [form.reset() method](https://developer.mozilla.org/en-US/docs/Web/API/HTMLFormElement/reset) or via [`<button type="reset" />` and `<input type="reset" />`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/reset).

`<inform-el />` also has a `reset()` which allows to specify new values while resetting.

<iframe width="100%" height="700" src="//jsfiddle.net/juliendelort/e1xujctm/16/embedded/html,result/" allowfullscreen="allowfullscreen" allowpaymentrequest frameborder="0"></iframe>

When the form is reset, the fields are considered "untouched" again, [which means that errors are hidden](/showing-errors.mdx).

The ["dirty" state](dirty-check.mdx) is also reset.

## Updating a value without reset

`<inform-el />` also has a `setValues()` method. It is identical to using reset above, expect that it doesn't reset the ["dirty" state](dirty-check.mdx), and the current errors remain displayed (fields stay "touched")"
